GitHub Branches
Post
Afgelopen week hadden Karel-Jan en ik het er kort over dat het direct op develop werken, in je eentje wel kan maar zodra je een ander persoon er ook aan komt te werken, het de boel alleen in de war schopt.

Ik was uit gemakzucht en omwille van de tijd die het telkens enorm veel tijd snoepte omdat ik niet exact wist wat ik aan het doen was…

Maar: vandaag op GitHub kwam ik het volgende tegen

https://odil.io/assets/blog/create-a-branch.png

hier werd ik wel blij van, dit betekend dat er vanuit GitHub de betreffende branch kan aangemaakt worden door Leo of iemand anders. Nu (heb ik in elk geval) geen gedoen meer dat er automagisch een branch wordt aangemaakt vanuit main in plaats van develop . Dit kwam omdat de default branch op main stond en niet op develop.

Na het klikken op het ‘Create a branch’-linkje krijg je het volgende venster:
https://odil.io/assets/blog/git-first-modal.png

Ik heb alvast de Branch Source op develop gezet in dit voorbeeld. Deze staat standaard op main. Hoewel je kan stellen, dan echte Bugs ook werken vanaf main dienen gebranched te worden.

Vanaf dat scherm is het vanzelfsprekend:
https://odil.io/assets/blog/git-cli-command.png

Bevestiging van het aanmaken, en meteen de commando’s voor het uitchecken.


Daarna zie je in het Issue onder de Assignment de nieuwe branch:
https://odil.io/assets/blog/branch-created.png

In dit geval volledig automatisch laten verlopen, hie zie je dat de titel van het Issue gepakt wordt voor de branch naam.
